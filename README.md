# Pattern Recognition Console App

**Pattern Recognition Console App** - це локальний консольний C++-проєкт для демонстрації простого розпізнавання образів за кількісними та якісними характеристиками. Програма порівнює новий об'єкт з еталонними класами та визначає, на який клас він найбільше схожий.

У роботі використано алгоритм класифікації за мінімальною евклідовою відстанню.

## Можливості Програми

- Описує об'єкти через вектор характеристик.
- Працює з кількісними ознаками: маса та розмір.
- Працює з якісною ознакою, закодованою числом.
- Обчислює евклідову відстань між новим об'єктом і кожним еталоном.
- Виводить відстань до кожного класу.
- Визначає найбільш схожий клас за найменшою відстанню.
- Підтримує український текст у консолі Windows через налаштування кодування `1251`.

## Класи Розпізнавання

У прикладі використано предметну область розпізнавання фруктів. Кожен фрукт описується трьома характеристиками:

- маса;
- розмір;
- якісна ознака форми.

Якісна ознака закодована так:

- `1` - кругла форма;
- `2` - видовжена форма;
- `3` - цитрусова форма.

Еталонні класи:

- `Яблуко` - `{150, 7, 1}`;
- `Банан` - `{120, 15, 2}`;
- `Апельсин` - `{200, 8, 3}`.

Новий об'єкт для розпізнавання:

```text
{140, 6, 1}
```

## Алгоритм

Для кожного еталонного об'єкта програма обчислює евклідову відстань:

```text
d(X, Y) = sqrt((x1 - y1)^2 + (x2 - y2)^2 + (x3 - y3)^2)
```

Після цього вибирається клас, для якого відстань є найменшою. Саме цей клас вважається результатом розпізнавання.

```text
ConsoleApp.cpp

  Файл рішення Visual Studio.

ConsoleApp.vcxproj
  Файл налаштувань C++-проєкту Visual Studio.

  Файл фільтрів для відображення структури проєкту у Visual Studio.
```

## Вимоги

Потрібно встановити:

- Visual Studio 2022 або новішу версію;
- компонент **Desktop development with C++**;
- Windows 10 або Windows 11.

Для запуску вже зібраного `.exe` достатньо Windows.

## Запуск У Visual Studio

1. Відкрити файл `ConsoleApp.sln`.
2. Вибрати конфігурацію `Debug` або `Release`.
3. Вибрати платформу `x64`.
4. Натиснути `Local Windows Debugger`.
5. Переглянути результат у консолі.

## Запуск З Термінала

Якщо виконуваний файл уже зібраний, його можна запустити з кореня проєкту:

```powershell
.\x64\Debug\ConsoleApp.exe
```

Після запуску програма одразу обчислює відстані до всіх еталонних класів і виводить результат розпізнавання.

## Збірка Проєкту

Проєкт можна зібрати через Visual Studio:

1. Відкрити `ConsoleApp.sln`.
2. Перейти в меню `Build`.
3. Натиснути `Build Solution`.

Після успішної збірки виконуваний файл буде створено в папці:

```text
x64\Debug\ConsoleApp.exe
```

## Приклад Результату

```text
Відстань до Яблуко: 10.0499
Відстань до Банан: 21.9545
Відстань до Апельсин: 60.0666

Результат: це найбільш схоже на -> Яблуко
```

Найменша відстань отримана для класу `Яблуко`, тому новий об'єкт класифікується як яблуко.

## Як Вносити Зміни В Програму

Змінити еталонні класи можна у файлі:

```text
ConsoleApp.cpp
```

Фрагмент з набором даних:

```cpp
vector<Object> dataset = {
    {"Яблуко", {150, 7, 1}},
    {"Банан",  {120, 15, 2}},
    {"Апельсин",{200, 8, 3}}
};
```

Змінити новий об'єкт для розпізнавання можна тут:

```cpp
vector<double> input = { 140, 6, 1 };
```

Після внесення змін потрібно перебілдити проєкт у Visual Studio.
